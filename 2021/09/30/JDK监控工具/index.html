常用工具命令

  1. jps (JVM Process Status）: 类似 UNIX 的 ps 命令。用户查看所有 Java 进程的启动类、传入参数和 Java 虚拟机参数等信息；
  2. **jstat**（ JVM Statistics Monitoring Tool）: 用于收集 HotSpot 虚拟机各方面的运行数据，用来检测垃圾回收和内存泄漏的问题
  3. jinfo (Configuration Info for Java) : Configuration Info for Java,显示虚拟机配置信息;
  4. jmap (Memory Map for Java) :生成堆【dump】存储快照文件，显示内存使用情况
  5. jhat (JVM Heap Dump Browser ) : 用于分析 dump 文件，它会建立一个 HTTP/HTML 服务器，让用户可以在浏览器上查看分析结果;
  6. jstack (Stack Trace for Java):生成虚拟机当前时刻的线程快照，线程快照就是当前虚拟机内每一条线程正在执行的方法堆栈的集合。
  7. jcmd：用来实现前面出jstat之外的所有功能

具体工具

jps

  1. 格式：jps [options] [hostid]
  2. jps -l：输出主类的全名，如果进程执行的是 Jar 包，输出 Jar 路径。
  3. jps -v：输出虚拟机进程启动时 JVM 参数，
  4. jps -m：输出传递给 Java 进程 main() 函数的参数【args】。
  5. jps -q：显示所有java进程的唯一id

jstat

  1. 格式：jstat -<option> [-t] [-h<lines>] <vmid> [<interval> [<count>]]
类装载
  1. jstat -class vmid ：显示 ClassLoader 的相关信息，类的装载，卸载数量，总空间
  2. jstat -class vmid 1000 10：没间隔1秒打印信息，总共打印10次
  3. jstat -class -t -h3 vmid：每个间隔3行就打印表头信息，t表示显示程序运行时长
垃圾回收

  1. jstat -gc vmid ：显示与 GC 相关的堆信息；包括新生代，老年代，方法区，已用空间

  2. jstat -gccapacity vmid ：显示各个代的容量及使用情况；

  3. jstat -gcnew vmid ：显示新生代信息；

  4. jstat -gccause vmid ：输出最后一次或当前GC产生的原因

  5. jstat -gcnewcapcacity vmid ：显示新生代大小与使用情况；

  6. jstat -gcold vmid ：显示老年代和永久代的行为统计，从jdk1.8开始,该选项仅表示老年代，因为永久代被移除了；

  7. jstat -gcoldcapacity vmid ：显示老年代的大小；

  8. jstat -gcpermcapacity vmid ：显示永久代大小，从jdk1.8开始,该选项不存在了，因为永久代被移除了；

  9. jstat -gcutil vmid ：显示垃圾收集信息；

  10. S0 S0C S0U S1 S1C S1U E O M CCS CCSC GCT LGCC
    幸存者0区的容量[collect] 已经使用的大小 压缩类的容量 ygc+fgc时间总和 发生gc的原因
  11. CCSU YGCT FGC FGCT EC EU OC OU MC YGC MU
    ygc时间 fgc次数 fgc时间 伊甸园区容量 老年代容量 方法区容量 ygc次数 方法区使用量
JIT相关的
  1. jstat -compiler vmid ：显示 JIT 编译耗时等相关信息
  2. jstat -printcompliation vmid：输出已经被JIT编译的方法

jinfo

  1. 格式：jinfo [option]
  2. jinfo -sysprops PID：展示当前程序进程的系统信息
  3. jinfo -flags [虚拟机参数] PID：查看所有赋过值的一些参数

jmap

  1. 格式：jmap [option] <pid>

  2. jmap -dump:live PID：生成该进程堆栈dump文件，加上live就只显示存活的对象

  3. jmap -heap PID：输出堆空间的详细信息

  4. jmap -histo PID：输出堆中对象的统计信息

  5. jmap -permstat PID：输出方法区的信息

  6. jmap -finalizerinfo PID：显示在F-Queue中等待finalizer线程执行finalize方法的对象

  7. jmap -F PID：

  8. jmp -dump:format=b,file=<fileName.hprof> <pid>：将生成的dump文件放在指定文件中

  9. jmp -dump:live,format=b,file=<fileName.hprof> <pid>：与上面的区别在于只有存活的对象

  10. 自动的方法：jvm参数

    • -XX：+HeapDumpOnOutOfMemoryError：在jvmOOM时生成dump文件
    • -XX：HeapDumpPath：指明dump文件的路径

jhat

  1. 格式： jhat [-stack <bool>] [-refs <bool>] [-port <port>] [-baseline <file>] [-debug <int>] [-version] [-h|-help] <file>，建议使用visualVM工具

jstack

  1. 格式：jstack [-l] <pid>，这是其一

jcmd

  1. jcmd -l：列出所有jvm进程
  2. jcmd pid 具体命令：对指定进程执行命令
